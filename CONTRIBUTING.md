# Contributing

Thanks for helping make TheGitReporter sharper, clearer, and more useful.

## Where To Contribute

- Use Discussions for story ideas, editorial critique, reader questions, and
  future guest-reporting workflow ideas.
- Use Issues for corrections, source challenges, broken links, bugs, and
  concrete public-site requests.
- Use Pull Requests for small public-site fixes and documentation improvements.

## Guest Articles

Guest article submissions are not open yet.

We like the direction: public pitches, public evidence, editorial review, then
publication through the private production pipeline. We are still polishing the
data flow and review boundaries before accepting article PRs.

Please discuss the future guest-reporter workflow in Discussions:

https://github.com/xbasset/thegitreporter/discussions

## Evidence Standard

When challenging or proposing a story, include public evidence when possible:

- commit URLs
- pull requests
- issues
- release notes
- documentation diffs
- API or runtime behavior changes

Make the claim easy to verify.

## Pull Request Scope

Good PRs for this repository include:

- typo fixes
- broken link fixes
- small documentation improvements
- public static-site metadata fixes

Please do not add private pipeline files, local cache output, credentials, raw
feedback exports, unpublished notes, or generated artifacts from another build
process.
