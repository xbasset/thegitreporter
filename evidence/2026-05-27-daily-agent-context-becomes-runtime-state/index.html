Reporter Notes

  • The most defensible thesis is not "memory is new" or "MCP is standardizing." It is narrower: context is becoming runtime state with protocol, storage, provenance, observability, and selection surfaces.
  • Codex supplies the densest evidence: additionalContext makes external context a typed turn input, memories_1.sqlite separates memory pipeline data from durable thread records, memory tool metrics make memory operations observable, and fork/session IDs make lineage groupable.
  • Gemini CLI broadens the arc with routing aliases for default roles. This is not memory, but it is a context-shaping decision moved into named configuration rather than hard-coded model choice.
  • Hermes broadens the arc through tool context: curated MCP entries and tool selection decide which external tool context can enter an agent session.
  • The article should keep "state > prompt" as an interpretation, not a literal implementation claim across all repos.

Primary Evidence

  • OpenAI Codex commit "Add experimental turn additional context (#24154)": https://github.com/openai/codex/commit/768848ab6fc6f05f19c447786fd76fd233fa1d69
  • Evidence used: adds experimental additionalContext support to turn start/steer calls, including separate untrusted and application kinds, hidden contextual message insertion, dedupe/reset behavior, and tests for the flow.
  • OpenAI Codex commit "Move memory state to a dedicated SQLite DB (#24591)": https://github.com/openai/codex/commit/aad59a0916baea1875f9a609a4b1d0a9062f24f6
  • Evidence used: moves memory-owned tables and jobs into a dedicated memories_1.sqlite runtime database while keeping thread metadata in the state database.
  • OpenAI Codex commit "Add ad-hoc memory note tool (#24562)": https://github.com/openai/codex/commit/3936ed221d90278a64d70a423fd7b456799f112b
  • Evidence used: adds a memories/add_ad_hoc_note tool behind the memories backend, validates append-only note filenames, rejects path-like names and empty notes, and keeps the surface behind existing registration controls.
  • OpenAI Codex commit "Add memory tool call metrics to memories extension (#24583)": https://github.com/openai/codex/commit/01a8bf0ae3e73d1697d2572a6666d5b47554f87f
  • Evidence used: adds a codex.memories.tool.call counter and tags memory tool calls by tool, operation, scope, status, and truncation state.
  • OpenAI Codex commit "Add forked_from_thread_id turn metadata (#24160)": https://github.com/openai/codex/commit/1911021c0eae972e622c8445d136ae04545a530f
  • Evidence used: adds turn metadata for fork lineage and protects Codex-owned lineage fields from turn-scoped client metadata overrides.
  • OpenAI Codex commit "[codex-analytics] add grouped session id to runtime events (#24655)": https://github.com/openai/codex/commit/487521733bea0a0a37559cffb519a1f6fb6bb444
  • Evidence used: adds session IDs to thread, turn, steer, compaction, and guardian review analytics so related root and subagent activity can be grouped.
  • Google Gemini CLI commit "Update default auto routing (#27071)": https://github.com/google-gemini/gemini-cli/commit/4a5d5cfc9f4baf314c966970a3041336a6ae2ec3
  • Evidence used: changes several documented roles from a fixed gemini-2.5-flash-lite model to a flash-lite routing alias and updates related CLI/model dialog tests.
  • NousResearch Hermes Agent commit "feat(mcp): Nous-approved MCP catalog with interactive picker (#30870)": https://github.com/NousResearch/hermes-agent/commit/8b69ec03af50de892ae0bca1f7e2384a8f6eb5a8
  • Evidence used: adds a curated MCP catalog, interactive picker, install command, optional MCP manifests, and install-time tool selection controls.

Evidence Limits

  • These sources show independent project changes during the same daily scan window; they do not prove coordination, a shared standard, or equivalent implementation choices.
  • The Codex memory note tool commit says the new tool surface is kept behind an existing registration path, so the article treats it as an implemented surface, not a broadly shipped user feature.
  • The Gemini CLI evidence supports a routing/configuration surface, not a claim about model quality or production rollout behavior beyond the cited commit.
  • The Hermes evidence supports a curated MCP catalog and picker in that repo, not an industry-wide MCP approval model.

Open Questions

  • Is Codex's additionalContext intended for long-term public clients or only experimental app-server consumers?
  • How soon, if ever, will the Codex ad-hoc memory note tool be exposed through app-server registration?
  • Does Gemini CLI's flash-lite alias select dynamically across models in all environments, or is it a named alias resolved by local policy/catalog code?
  • Will curated MCP catalogs converge across projects, or remain repo-specific trust lists?
