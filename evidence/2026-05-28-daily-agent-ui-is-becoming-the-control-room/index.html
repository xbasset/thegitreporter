Reporting Notes

The strongest story today is not "agent tools got nicer UIs." The source-grounded version is more specific: supervision now needs handles. Codex adds link handles in terminal output and interruption cleanup handles in the shell runtime. Hermes adds session handles. Gemini CLI removes one class of accidental normal-mode key leakage while keeping modifier shortcuts available.

This is a natural continuation of the runtime-state sequence but should read more human than infrastructure-heavy. The reader-facing point is that an operator needs to click evidence, stop work, switch between live threads, and trust keyboard behavior before long-running agents feel controllable.

Claim Boundaries

  • Do not say these projects coordinated.
  • Do not say this is a standard.
  • Do not say the OSC 8 work makes every terminal link clickable; the commit specifically depends on terminals that support OSC 8 links.
  • Do not overstate the Gemini CLI fix. It is a targeted input-handling improvement.
  • Do not turn interruption cleanup into a general safety guarantee. The commit addresses a specific shell-command cancellation race and cleanup path.

Primary Evidence

  • OpenAI Codex commit "feat(tui): add OSC 8 web links to rich content (#24472)": https://github.com/openai/codex/commit/7a26497836be29ec77fb8007347b33cad952b497
  • Evidence used: adds OSC 8 web-link support for rich TUI output, validates web destinations, preserves complete URL targets through wrapping and replay paths, and routes Codex-owned notices/help surfaces through shared hyperlink handling.
  • OpenAI Codex issue context referenced by the OSC 8 commit: https://github.com/openai/codex/issues/12200#issuecomment-4535452980
  • Evidence used: public context for the semantic-link problem that the OSC 8 commit says it addresses.
  • OpenAI Codex commit "fix(linux-sandbox): preserve shell cleanup on interruption (#22729)": https://github.com/openai/codex/commit/9152ebd289f2d9103ffc41c2fec69f6c623a6eab
  • Evidence used: changes shell-command cancellation so interrupted Linux sandbox work can run cleanup before hard termination, and adds regression coverage around cleanup-on-cancellation behavior.
  • NousResearch Hermes Agent commit "feat: add TUI session orchestrator": https://github.com/NousResearch/hermes-agent/commit/0a83247e9fa6fc0d94ebbd05626b666c520c3d43
  • Evidence used: adds a TUI active-session orchestrator that can list, activate, close, and launch live sessions, hydrate committed and in-flight output, and expose a clickable live-session count.
  • Google Gemini CLI commit "fix(cli): ignore unmapped vim normal keys (#27102)": https://github.com/google-gemini/gemini-cli/commit/5cac7c10fa9ff34e99553057631727c95c1e99f8
  • Evidence used: adjusts Vim normal-mode input handling so unmapped printable keys are consumed while modifier chords can still fall through, with regression tests for several keys.

Evidence Limits

  • These sources show independent project changes; they do not prove coordination, a shared UI standard, or equivalent implementation choices.
  • The evidence supports a pattern in agent supervision interfaces, not a claim that any one product has fully solved long-running agent oversight.
  • The Hermes source shows a project-specific TUI session orchestrator, not broad availability across all Hermes deployments.
  • The Gemini CLI source is a narrow keyboard-handling fix. It is used as a small supporting signal, not as proof of a large UI redesign.
