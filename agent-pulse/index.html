Before a daemon passes identity to tools, prove who may receive it

Qwen Code added a narrow invocation-context path from a daemon-admitted root prompt to Qwen-launched local stdio MCP tools. The source carries a session-and-prompt label for correlation while separately scrubbing the private capability that authenticated the daemon child from tool environments.

Today's Action

Audit / Project-specific claim
action

Before a daemon passes identity to tools, prove who may receive it

What changed. The project reconstructs reserved metadata from daemon state, binds it to one root turn, and sends it only over eligible stdio transports. Its design excludes background and subagent paths and non-stdio transports; an end-to-end test observes the metadata at a local echo tool while checking that the private capability is absent.

Why it matters. A shared agent runtime can make a tool call traceable without making every downstream tool a holder of the authority that admitted the work. Separating correlation from credentials gives builders a concrete place to test transport scope, turn lifetime, and secret non-propagation.

Claim coverage / complete 24 / 24

All 24 included projects and all 13 core coding agents have fresh source receipts for this edition.

Agent-platform builder 30–45 minutes per transport

Map the handoff boundary

For each daemon-to-tool path, write down the identity fields a tool needs and the capability it must never receive. Rebuild reserved metadata at the trusted admission boundary instead of forwarding caller-supplied fields.

Reversible
Agent-platform tester 60–90 minutes per runtime path

Test both halves of the contract

Run one root prompt through an eligible local tool and assert the expected correlation record. In the same test, inspect the tool environment and non-eligible transports to prove the private token and metadata do not leak there.

Reversible
Qwen Code operator 20–30 minutes

Treat this as source evidence, not deployment evidence

Before relying on provenance metadata in a shared-session workflow, verify the release you use contains the change and run a non-sensitive local MCP test that captures what the tool receives.

Reversible

Shows the source-level invocation-context path, transport plumbing, private-capability sanitization, design note, and end-to-end regression coverage that verifies metadata arrives at a local tool while the private capability does not.

Confidence / strong

The public commit, scoped design contract, implementation, and end-to-end local MCP test agree on the intended handoff and secret boundary; the claim stays confined to Qwen Code’s source change.

Limit

This is one project’s source-level contract. It does not establish that a release or deployment includes the feature, that every MCP server handles metadata safely, or that the design proves a general security property across other agent runtimes.

What would change this

Revise this call if later source shows the reserved metadata is caller-controlled, if a non-eligible transport receives it, if the private capability reaches a tool environment, or if the change is removed before release.

Watch next. Watch for release inclusion and a public operator test that exercises the boundary with a real shared session, an eligible local tool, and a non-eligible transport.
