A cloud coding command makes repository selection part of the agent boundary

Kilo Code added a cloud-agent command that builds a remote task from the current Git worktree or an explicit repository. The source validates the repository and branch before it builds the request, treating source selection as a control surface rather than a background convenience.

Today's Action

Watch / Project-specific claim
watch

A cloud coding command makes repository selection part of the agent boundary

What changed. The new command resolves GitHub shorthand, standard GitHub SSH URLs, or validated HTTPS sources; rejects credentials, ambiguous remotes, unsafe branches, and non-worktrees; then sends a structured task with a selected repository, plan-mode agent, account context, and queued delivery. Focused tests exercise those request and failure paths.

Why it matters. When a local CLI can hand code to a remote agent, the repository reference and branch tell the service what it may inspect. Builders should make that handoff visible and testable before treating a cloud path as equivalent to a local command.

Claim coverage / complete 24 / 24

All 24 included projects and all 13 core coding agents have fresh source receipts for this edition.

Agent-platform builder 45–60 minutes per remote-execution command

Audit the remote repository handoff

For every command that starts remote work from a local checkout, test an ambiguous remote, a credential-bearing URL, an unsafe branch, and a non-worktree. Confirm the request fails before any remote task is created.

Reversible
Kilo Code evaluator 20–30 minutes

Watch one real task boundary

Before relying on `kilo cloud` for a sensitive repository, inspect the shipped release and run a non-sensitive test repository first. Compare the selected repository, branch, task status, and cancellation behaviour with the source contract.

Limit

What would change this

Revise this call if later source or tests show that repository validation happens after task creation, if a supported remote form bypasses the stated checks, or if the feature is removed before release.
