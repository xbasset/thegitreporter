API Discovery: Recherche d'entreprises Turns Company Search Into Agent Triage

API Recherche d'entreprises exposes public French company search through text, SIREN/SIRET, geography, NAF, and employee-size filters.

repos source trail evidence
4 source signals 0 repos 4 source signals

Open Edition Evidence below

The public source is api.gouv.fr's catalogue file for API Recherche d'entreprises and its linked OpenAPI definition. The /search endpoint accepts a company name, address, executive or elected-official query, or direct SIREN/SIRET lookup, with filters such as postal code and principal activity.

This belongs in a compact slot, not the lead, because the catalogue did not show a same-day product change; the useful decision is narrow. Operators can still use it before outreach, onboarding, or vendor review to make an agent verify the company record it is about to mention.

Before an agent drafts a supplier email, it can search the company name, confirm the SIREN/SIRET candidate, check address or postal-code fit, and ask for human review when multiple legal units match.

Operators can paste this into an agent:

Use API Recherche d'entreprises. Query the company name, then if a SIREN or SIRET appears in the source document, run a direct lookup. Return the likely legal unit, SIREN/SIRET, address, NAF code, uncertainty, and one reason a human should verify before outreach.

The catalogue lists a 7 calls-per-second limit and says direct SIREN/SIRET search ignores other filters. The next useful signal would be a catalogue update with fresher monitoring, richer fields, or clearer ambiguity handling for agent workflows.
