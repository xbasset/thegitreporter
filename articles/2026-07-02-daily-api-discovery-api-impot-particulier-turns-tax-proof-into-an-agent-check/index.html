API Discovery: API Impot Particulier Turns Tax Proof Into An Agent Check

France's API.gouv catalog exposes a DGFiP tax-data API where agents must treat authorization and proportionality as the product, not an afterthought.

repos source trail evidence
4 source signals 0 repos 4 source signals
Evidence: 4 source signals / July 2, 2026 / Daily Edition
Open Edition Evidence below

API.gouv's API Impot particulier page and OpenAPI definition show a restricted DGFiP source for fiscal-reference data, tax-assessment situations, housing-tax data, and LEP eligibility checks.

The creative agent use case is a benefits intake assistant that checks whether a public-service workflow can replace a tax-document upload with an authorized API lookup, then stops and asks for a human eligibility decision before any request is made.

Copy-Paste Agent Instruction

Review API Impot particulier for my public-service workflow. Identify the exact endpoint family I would need, the legal eligibility condition, the minimal data fields, and the DataPass or authorization step. Do not draft code that calls the API until I confirm my organization is eligible.

Implication

For governance teams, the point is not speed alone. The catalog page says access is restricted, asks administrations to justify citizen simplification and proportionality, and routes banks toward a separate SFiP path for LEP eligibility.

Test Caveat

This is not a public unauthenticated API to try from a prompt. What to watch: whether API.gouv keeps pairing OpenAPI files with eligibility language clear enough for agents to recommend the right human approval path.

Primary Evidence

  • API.gouv catalog page for API Impot particulier: https://github.com/betagouv/api.gouv.fr/blob/master/_data/api/impot-particulier.md
    • Shows: the API is restricted, produced by DGFiP, tied to tax data, FranceConnect/user identification, and eligibility/authorization language.
  • OpenAPI definition for API Impot particulier: https://github.com/betagouv/api.gouv.fr/blob/master/public/swaggers/api-impot-particulier.json
    • Shows: endpoint families for income-tax situations, tax-assessment data, housing-tax data, and LEP eligibility checks.
  • DGFiP producer page in API.gouv: https://github.com/betagouv/api.gouv.fr/blob/master/_data/producteurs/dgfip.md
    • Shows: the producer context for French fiscal data.
  • API.gouv LEP guide: https://github.com/betagouv/api.gouv.fr/blob/master/_data/guides/livret-epargne-populaire.md
    • Shows: the banking LEP use case is routed through SFiP and requires an authorization path.

Evidence Limits

  • This is not a public unauthenticated API. The source supports a governance-first agent instruction, not a test call.
  • What to watch: whether API.gouv keeps endpoint definitions, eligibility language, and DataPass routes aligned for agent-assisted public-service workflows.
