API Discovery: Check a Barcode Against France's Recall List

France's official RappelConso data can turn a scanned GTIN into a product-recall match—if the agent checks the lot, date, and government notice instead of treating a name hit as proof.

API discovery: A live query to the French government's RappelConso Explore API for barcode 3701002416628 returned one July 10 recall: a 90-gram aperitif product, with its lot, durability date, stated listeria risk, instructions not to consume it, and a link to the official recall sheet.

Try it on a receipt

Give an agent the barcodes from a household inventory or shopping receipt. It can query each identifier, flag an exact recall candidate, and return the brand, model, affected lot or date, official action, and government notice. The useful output has three states: exact record to verify, possible name match needing identifiers, or no match found in this query.

Copy-paste agent instruction

Use only the official RappelConso dataset at https://data.economie.gouv.fr/api/explore/v2.1/catalog/datasets/rappelconso-v2-gtin-espaces/records. For each barcode I give you, query where=search(*,"BARCODE") and request the product, identification, risk, consumer-action, publication-date, and official-notice fields. Require an exact GTIN match, then compare the lot, model, and date before flagging it. Link the official recall sheet. If nothing matches, say "no match found in this query"—never "safe." Do not infer a match from the brand alone.

Test caveat

The sample query worked without authentication on July 11, but this is a screening workflow, not a safety oracle. Records, rate policy, and availability can change; product identity may depend on a lot or date that a barcode does not encode. A miss is not proof of safety, and every hit should end at the current official notice.

Primary Evidence

  • Public API source: https://github.com/betagouv/api.gouv.fr/blob/master/_data/api/api-rappel-conso.md
    • Shows: DGCCRF is the producer and publishes the open RappelConso API dataset with product, risk, location, and consumer-action fields.
  • Supporting definition or documentation: https://data.economie.gouv.fr/explore/assets/rappelconso-v2-gtin-espaces/api/
    • Shows: the public records endpoint, query surface, and current dataset fields.
  • Reproducible GTIN query for 3701002416628: https://data.economie.gouv.fr/api/explore/v2.1/catalog/datasets/rappelconso-v2-gtin-espaces/records?where=search%28%2A%2C%223701002416628%22%29&select=date_publication%2Cnumero_fiche%2Cmarque_produit%2Cmodeles_ou_references%2Cidentification_produits%2Cmotif_rappel%2Crisques_encourus%2Cconduites_a_tenir_par_le_consommateur%2Clien_vers_la_fiche_rappel&limit=5
    • Shows: one exact July 10 recall result with the product identity, GTIN, lot/date data, stated risk, consumer action, and official notice URL.
  • Official recall sheet returned by the query: https://rappel.conso.gouv.fr/fiche-rappel/22807/interne
    • Shows: the government notice that a user should inspect before acting.

Evidence Limits

  • An API match is a lead to verify against the exact product, lot, date, and official notice; it is not a diagnosis or an automatic safety decision.
  • An empty query result means only that the request found no matching record. It must never be presented as proof that a product is safe.
  • The endpoint answered without a key in the July 11 test, but access, rate limits, availability, update latency, and record completeness can change.
